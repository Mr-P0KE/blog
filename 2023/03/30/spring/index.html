三、BeanFactoryPostProcessor（需要注入容器）

常见的bean工厂后处理器 区别 继承关系 作用 执行顺序
BeanFactoryPostProcessor（接口） 不带注册功能 顶级接口 补充beanDefinition 2
BeanDefinitionRegistryPostProcessor（接口） 带注册功能 继承BeanFactoryPostProcessor（次顶级接口） 添加beanDefinition 1（带注册的方法、然后执行不带注册的方法）

1.两者源码

public interface BeanFactoryPostProcessor {
    void postProcessBeanFactory(ConfigurableListableBeanFactory var1) throws BeansException;
}
//===========================
public interface BeanDefinitionRegistryPostProcessor extends BeanFactoryPostProcessor {
    void postProcessBeanDefinitionRegistry(BeanDefinitionRegistry var1) throws BeansException;
}

使用

@Comment
public class MyBeanDefinitionRegistryPostProcessor implements BeanDefinitionRegistryPostProcessor {
    @Override
    public void postProcessBeanFactory(ConfigurableListableBeanFactory beanFactory) throws BeansException {
        for(String name:beanFactory.getBeanDefinitionNames()) {
            if("lifeCycleBean".equals(name)) {
                log.debug("BeanDefinitionRegistryPostProcessor-->bean工厂后置处理器>>>>>>> bean定义之后，实例化之前执行,对bean做一些改变");
            }
        }
    }

    @Override
    public void postProcessBeanDefinitionRegistry(BeanDefinitionRegistry beanDefinitionRegistry) throws BeansException {
        for(String name:beanDefinitionRegistry.getBeanDefinitionNames()) {
            if("lifeCycleBean".equals(name)) {
                log.debug("BeanDefinitionRegistryPostProcessor-->bean工厂后置处理器>>>>>>> bean定义之后，实例化之前执行，注册更多的bean到容器");
            }
        }
    }
}

1.代理模式

1.1 jdk代理

1.1.1 jdk代理的简单实现
public class JdkProxy {
    interface Foo{
        void m();
    }
    static class Target implements Foo{
        @Override
        public void m() {
            System.out.println("方法执行");
        }
    }

    public static void main(String[] args) {
        Target target = new Target();
        Foo o =(Foo) Proxy.newProxyInstance(JdkProxy.class.getClassLoader(),
                Target.class.getInterfaces(),
//                new InvocationHandler(参数...) 的Lambda表达式写法
                 (proxy, method, args1) -> {
                    System.out.println("执行前");
                    Object o1 = method.invoke(target, args1);
                    return o1;
                });
        o.m();
    }
}
1.1.2 JDK代理在spring中的模拟实现
public class Main {
    //接口，被代理对象
    interface Foo{
        void m() throws NoSuchMethodException, InvocationTargetException, IllegalAccessException;
        int s() throws NoSuchMethodException, InvocationTargetException, IllegalAccessException;
    }
    //被代理对象
    static class Target implements Foo {
        @Override
        public void m() {
            System.out.println("方法执行mmmm");
        }

        @Override
        public int s() {
            System.out.println("方法执行ssss");
            return 100;
        }
    }
    //接口，确定调用的方法
    interface InvocationHandler{
        Object invoke(Object proxy,Method method, Object[] args) throws InvocationTargetException, IllegalAccessException;
    }

    public static void main(String[] argss) throws NoSuchMethodException, InvocationTargetException, IllegalAccessException {
        $Proxy0 proxy0 = new $Proxy0((proxy, method, args) ->  {
            System.out.println("功能增强");
            //反射
            return method.invoke(new Target(), args);
        });
        proxy0.m();
        System.out.println(proxy0.s());
    }
//模拟：代理对象        有点像装饰器模式
public class $Proxy0 extends Proxy implements Main.Foo{
    static Method m;
    static Method s;
    static {
        try {
             m = Main.Foo.class.getMethod("m");
             s = Main.Foo.class.getMethod("s");
        } catch (NoSuchMethodException e) {
            throw new NoSuchMethodError(e.getMessage());
        }
    }
    
    public $Proxy0(InvocationHandler h) {
        super(h);
    }

    @Override
    public void m() throws Throwable {
        h.invoke(this,m,new Object[0]);
    }

    @Override
    public int s() throws Throwable {

        return (int) h.invoke(this,s,new Object[0]);
    }
}

八、事务

1.概念

事务传播行为用来描述由某一个事务传播行为修饰的方法被嵌套进另一个方法的时事务如何传播。

2.事务传播行为

传播属性 描述
Required 如果当前存在事务，则加入该事务，如果当前不存在事务，则创建一个新的事务。
Requied_new 重新创建一个新的事务，如果当前存在事务，延缓当前的事务。
supports 如果当前存在事务，则加入该事务；如果当前不存在事务，则以非事务的方式继续运行。
no_supported 以非事务的方式运行，如果当前存在事务，暂停当前的事务。
mandatory 如果当前存在事务，则加入该事务；如果当前不存在事务，则抛出异常。
Never 以非事务的方式运行，如果当前存在事务，则抛出异常。
Nested 如果没有，就新建一个事务；如果有，就在当前事务中嵌套其他事务。
事务传播行为类型 说明
PROPAGATION_REQUIRED 如果当前没有事务，就新建一个事务，如果已经存在一个事务中，加入到这个事务中。这是最常见的选择。
PROPAGATION_SUPPORTS 支持当前事务，如果当前没有事务，就以非事务方式执行。
PROPAGATION_MANDATORY 使用当前的事务，如果当前没有事务，就抛出异常。
PROPAGATION_REQUIRES_NEW 新建事务，如果当前存在事务，把当前事务挂起。
PROPAGATION_NOT_SUPPORTED 以非事务方式执行操作，如果当前存在事务，就把当前事务挂起。
PROPAGATION_NEVER 以非事务方式执行，如果当前存在事务，则抛出异常。
PROPAGATION_NESTED 如果当前存在事务，则在嵌套事务内执行。如果当前没有事务，则执行与PROPAGATION_REQUIRED类似的操作。

3.注解声明事务
