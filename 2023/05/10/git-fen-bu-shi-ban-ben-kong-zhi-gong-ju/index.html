Git分布式版本控制工具

1.为什么需要Git？

  • 备份
  • 代码还原
  • 协同开发
  • 追溯代码编写人和编写代码的时间

2.版本控制器的方式

  • 集中式版本控制工具
    • 版本库集中存放在中央服务器，team中每个人在工作时，都需要从中央服务器下载代码，必须联网（局域网，互联网），个人修改后提交到中央版本库。
    • SVN,CVS
  • 分布式管理控制工具
    • 没有中央服务器，每个电脑都是一个完整的版本库，工作时不需要联网。版本库在自己电脑上，多人协作只需要将代码推送对方（很少）。
    • Git

3.工作流程

总流程：

分步流程：

本地仓库：

*工作区 -> 暂存区
git add 文件名
git add .  #添加所有文件到暂存区
*暂存区 -> 本地仓库
git commit 
git commit -m "add xx.xx"  #设置备注
*查看状态
git status
**查看提交日志
  • 命令形式：git log [option]
  • options
    • –all 显示所有分支
    • –pretty=oneline 将提交信息显示为一行
    • –abbrev-commit 让输出的commitID更简短
    • –graph 以图的形式展示
git log
*查看所有日志
git relog

4.基本配置

  • 打开Git Bash（就是右键打开嘛）

  • 设置用户信息

    • git config –global user.name “name”
    • git config –global user.email “email”

  • 查看配置信息

    • git config –global user.name
    • git config –global user.email

5.获取本地仓库

git init

6.查看文件状态

git status

*7.版本控制

命令形式：

git reset --hard commitID(使用log日志查看)

8.设置忽略的文件

  • 创建 .gitignore 文件
  • 里边设置文件或 &.a 设置后缀

*分支

查看分支、创建分支

git branch        #查看分支
git branch 分支名  #创建分支

切换分支

git checkout 分支名  #切换分支

创建并切换分支

git checkout -b 分支名

*合并分支

git merge 分支名

删除分支

git branch -d 分支名 
git branch -D 分支名 #强制删除

*合并分支出现的冲突问题

  • 确定自己想要的

  • add .

  • commit

分支名规范定义：

  • master : 生产分支
  • develop: 开发分支
    • 从master 创建的分支，开发部门使用，开发完成合并master
  • feture/xxx ：
    • 从develop创建的分支，一般同时并行开发，但是上线时不同，开发完成合并develop分支。
  • hotfix/xxx
    • 从master派生的分支，一般用于bug修复，修复完成合并到上面三个分支。
