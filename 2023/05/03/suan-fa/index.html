举例：二分查找

没啥要说的，上代码，肝！

问题版本：


 public int method1(int[] nums,int target){
        int l = 0,r = nums.length-1,mid = 0;
      // l < r 会出现-1的情况：原因：l = r 不会运行判断
        while(l <= r){
            mid = (r - l) /2 +l;
//             mid = (r + l ) / 2;
            if(nums[mid] == target){
                return mid;
            }else if(nums[mid] < target){
                l = mid + 1;
            }else
                r = mid - 1;

        }
        return -1;
    }

改进版本：

问题：

mid = (r + l ) / 2;有什么问题？

计算机底层使用二进制数，最高位为符号位。（符号位参与计算）

可能超过int类型范围，造成溢出（r = Integer.MAX_VALUE 在 重新计算左边界 L，L会溢出。（符号位参与计算））

解决方法：

使用无符号数右移。（>>> 1）

public int method2(int[] nums,int target){
        int l = 0,r = nums.length-1,mid = 0;
        while(l <= r){
           mid = (l+r) >>>1;
            if(nums[mid] == target){
                return mid;
            }else if(nums[mid] < target){
                l = mid + 1;
            }else
                r = mid - 1;

        }
        return -1;
    }

进阶版本：

public int method3(int[] nums,int target){
        int l = 0,r = nums.length,mid = 0;
       //改动，因为r取不到，（l <= r，可能陷入死循环）
        while(l < r){
            mid = (l+r) >>> 1;
            if(nums[mid] == target)
            return mid;
            else if(nums[mid] < target)
                //改动，因为l可以取到
            l = mid+1;
            else
            r = mid ;
        }
        return -1;
    }

时间复杂度：

最坏: $O(logn)$

最好：$O（1）$

空间复杂度： $O(1)$

平均版本：

目的：减少平均比较次数。(大数据量)

算法的好坏判断：

时间复杂度： $O(logn)$

渐进上界：从某个常数n开始，c*g(n)总位于f(n),上方，记作O（g（n））

eg : f(n) = 3*n+3

​ g(n) = n ==== > O(n)

    • $$
      f(n) =
      \left{
      \begin{matrix}

      \right.
      $$

public static String reverse(String s,int length) {

public static int search(int[] a,int target){
      return search1(a, target, 0, a.length-1);
    }
public static int search1(int[] a,int target,int l,int r){
        if(l > r)
        return -1;
        int mid = (l+r) >>>1;
        if(a[mid] == target)
        return mid;
        else if(a[mid] < target)
        return search1(a, target, mid+1, r);
        else 
        return search1(a, target, l, mid-1);
    }

2.斐波那契数列的兔子问题

个人想出来的方法：
//规律 ： 每个月可生产的兔子为 ：（n-1）/2
public static int ff(int n){
        return f1(n,(n-1)/2);
    }
    public static int f1(int n,int m){
        if(n == 1 || n == 2)
        return 2;
        int sum =  f1(--n, (n-1)/2);
        return sum += 2 * m;
    }
标准方法：

$$
f(n) =
\left{
\begin{matrix}
f(1) = 2~~~~结束条件\
f(2) = 2~~~~结束条件\
f(n) = f(n-1)+f(n-2)\

    \end{matrix}
\right.

$$

分析(个人理解)：

n个月的兔子数 f ( n ) = 可生产的兔子数 + 不可生产兔子数 + 生产出来的兔子数量

可生产的兔子数量 = n-2 的兔子数量，f ( n - 2)

不可生产的兔子数量 = f (n -1) - f ( n - 2)

生产出来的兔子数量 = f ( n - 2 )

f ( n ) = f ( n - 2) + f (n -1) - f ( n - 2) + f ( n - 2 ) = f ( n - 2) + f (n -1)

公式：
$$
f(n) =
\left{
\begin{matrix}
1 ~~~~~~~~~~~~~~~n =1,n=2\
f(n-1)+f(n-2)~~~~~ n>2\
\end{matrix}
\right.
$$

简单题，直接上代码！！！

public static int method(int n) {
       if(n == 1 || n == 2)
       return 1;
       return  method(n-1) + method(n-2);
   }

public void loop(){
    Node temp = head;
    recursion(temp);
}
/**
* @Author Mr_Poke
* @Date  2023/5/5
* @Description  递归实现方法
* @Param  * @param head
* @Return void
*/
private void recursion(Node head){
    if(head == null)
        return ;
     recursion(head.next);
}

使用Consumer接口（消费型接口）的显示节点数值：

public void loop(Consumer<Integer> before,
                     Consumer<Integer> after){
        Node temp = head;
        recursion(temp,before,after);
    }
    /**
    * @Author Mr_Poke
    * @Date  2023/5/5
    * @Description  递归实现方法
    * @Param  * @param head
    * @Return void
    */
    private void recursion(Node head,Consumer<Integer> before,
                           Consumer<Integer> after){
        if(head == null)
            return ;
        before.accept(head.val);
        recursion(head.next,before,after);
        after.accept(head.val);
    }
