位运算

正数：在内存中以原码存放，原码 = 反码 = 补码（三码合一）

负数：在内存中以补码存放

什么是原码，反码，补码？

原码： 一个数的二进制数。

反码：正数的反码 = 原码，负数的反码 = 原码按位取反（除去符号位）

补码：正数的补码 = 原码，负数的补码 = 反码+1

-5
原码： 1000 0000 0000 0000 0000 0000 0000 0101
反码： 1111 1111 1111 1111 1111 1111 1111 1010
补码： 1111 1111 1111 1111 1111 1111 1111 1011

1.&运算（按位与）

按位取，有0则0，（两个数都为1才为1）

void main() {
    printf("%x\n",-5 & 10); // a（10）
    /*
     *  -5（反码） : 1111 1111 1111 1111 1111 1111 1111 1011
     *  10 : 0000 0000 0000 0000 0000 0000 0000 1010
     *与运算: 0000 0000 0000 0000 0000 0000 0000 1010 = a(10)
     */
}

2.|运算（按位或）

按位取，有1则1，（有一个数位1就为1）

void main() {
    printf("%x\n",-5 | 10); // -5
    /*
     *  -5（反码） : 1111 1111 1111 1111 1111 1111 1111 1011
     *  10 : 0000 0000 0000 0000 0000 0000 0000 1010
     
     *或运算: 1111 1111 1111 1111 1111 1111 1111 1011 = fffffffb
     * *或运算的反码: 1000 0000 0000 0000 0000 0000 0000 0100
     *        补码： 1000 0000 0000 0000 0000 0000 0000 0101 = -5
     */
}

3.~运算（按位取反）

按位取反，1 - > 0,0 -> 1

void main() {
    printf("%d\n",~-5);
    /*
    -5（反码）: 1111 1111 1111 1111 1111 1111 1111 1011
    ~运算 ：    0000 0000 0000 0000 0000 0000 0000 0100 = 4
    */
    }

4.^运算（按位异或）

相异得1，相同得0

printf("%d\n",-5^10);
/**按位异或： 1111 1111 1111 1111 1111 1111 1111 0001
     * 反码：   1000 0000 0000 0000 0000 0000 0000 1110 
     * 补码：   1000 0000 0000 0000 0000 0000 0000 1111 = -15
     */

5.位移

左移<<（乘2的n次方，低位溢出，有误差）

右移>>（除2的n次方，低位溢出，有误差）

分为逻辑右移、算数右移（有符号数）

逻辑右移：高位补0，低位溢出

eg：90 = 0101 1010 >>3

0000 1011 = 11

算数右移：高位补符号位，低位溢出（有符号数）

eg： -45 = 1010 1101>>3

-117 = 1111 0101

加减乘除

加法（异或+与+左移+异或+与（结果不0，继续异或））

举个例子：求4+5？

计算机是如何操作的呢？
0000 0100
0000 0101
-------------------------（加法，计算机是不会直接加）
0000 1001


那么计算机的实现原理是什么呢？
怎么将两个数加起来？核心是：异或。

第一步，异或（不一样为1）：如果不考虑进位，异或就可以直接出结果
0000 0100
0000 0101
-------------------------
0000 0001


第二步，计算机在这个异或的结果上在做与运算操作：
与运算（判断进位），如果与运算结果为0，那么就没有进位。
0000 0100
0000 0101
-------------------------
0000 0100


第三步，将与运算的结果左移一位。
0000 0100 ——> 0000 1000 (进位的结果)


第四步，还是一个异或运算。（第一步异或出来的结果和第三步与运算的进位结果再一次异或）
0000 0001
0000 1000
------------------------
0000 1001


第五步，再去做一个与运算，判断它是否有进位。（与第二步一样）
0000 0001
0000 1000
------------------------
0000 0000


最后一步与运算结果为
0000 0000
电路都断了，灯泡全灭，通过不了，所以最终结果就是：
与运算为0的结果的上一个异或运算结果：
0000 1001 （二进制的9）


如果与运算不为0，继续循环。

4-5 说白了就是 4+(-5)

1111 1011 (代表二进制的-5)

1111 1111（8个1就是ff,所以ff在十进制中代表-1）

------------------------ 异或(如果不考虑进位，异或就可以直接出结果)
1111 1111


0000 0100
1111 1011
------------------------ 与运算（判断进位），如果与运算结果为0，那么就没有进位。
0000 0000


最终结果
1111 1111 (十六进制的ff，十进制的-1)

乘法： x * y，本质就是y个x相加，还是加法。
除法： x / y，本质就是减法，就是x能减去多少个y。

除法：

除数是2的n次方

除数是2的n次方，则

商 = 被除数 >>> n

余数的 = 被除数的右移的值

计算余数： 被除数 ^ ($2^n$ -1)

111 /  8 = 13  余 7
被除数：  0110 1111  = 111
去除数：  0000 1000  =  8
----------------
去商值：  0000 1101  =  13（右移三位）
去余数：  0000 0111  =   7

=========================

111 / 16 = 6  余  15
被除数：  0110 1111  = 111

----------------
