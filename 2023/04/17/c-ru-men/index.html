1.register（寄存器）

修饰的变量寄存器变量。

即：编译时告诉编译器变量是寄存器变量，尽量将内存空间分配在寄存器中

注意！

  1. 不一定存在寄存器中。
  2. cpu取寄存器中的数据 比在 内存中取 快
  3. 不能定义数组
  4. 不能修饰浮点型.
  5. 放在寄存器中，无法对变量取地址（指针）

位运算

正数：在内存中以原码存放，原码 = 反码 = 补码（三码合一）

负数：在内存中以补码存放

什么是原码，反码，补码？

原码： 一个数的二进制数。

反码：正数的反码 = 原码，负数的反码 = 原码按位取反（除去符号位）

补码：正数的补码 = 原码，负数的补码 = 反码+1

-5
原码： 1000 0000 0000 0000 0000 0000 0000 0101
反码： 1111 1111 1111 1111 1111 1111 1111 1010
补码： 1111 1111 1111 1111 1111 1111 1111 1011

1.&运算（按位与）

按位取，有0则0，（两个数都为1才为1）

void main() {
    printf("%x\n",-5 & 10); // a（10）
    /*
     *  -5（反码） : 1111 1111 1111 1111 1111 1111 1111 1011
     *  10 : 0000 0000 0000 0000 0000 0000 0000 1010
     *与运算: 0000 0000 0000 0000 0000 0000 0000 1010 = a(10)
     */
}

2.|运算（按位或）

按位取，有1则1，（有一个数位1就为1）

void main() {
    printf("%x\n",-5 | 10); // -5
    /*
     *  -5（反码） : 1111 1111 1111 1111 1111 1111 1111 1011
     *  10 : 0000 0000 0000 0000 0000 0000 0000 1010
     
     *或运算: 1111 1111 1111 1111 1111 1111 1111 1011 = fffffffb
     * *或运算的反码: 1000 0000 0000 0000 0000 0000 0000 0100
     *        补码： 1000 0000 0000 0000 0000 0000 0000 0101 = -5
     */
}

3.~运算（按位取反）

按位取反，1 - > 0,0 -> 1

void main() {
    printf("%d\n",~-5);
    /*
    -5（反码）: 1111 1111 1111 1111 1111 1111 1111 1011
    ~运算 ：    0000 0000 0000 0000 0000 0000 0000 0100 = 4
    */
    }

4.^运算（按位异或）

相异得1，相同得0

printf("%d\n",-5^10);
/**按位异或： 1111 1111 1111 1111 1111 1111 1111 0001
     * 反码：   1000 0000 0000 0000 0000 0000 0000 1110 
     * 补码：   1000 0000 0000 0000 0000 0000 0000 1111 = -15
     */

5.位移

左移<<（乘2的n次方，低位溢出，有误差）

右移>>（除2的n次方，低位溢出，有误差）

分为逻辑右移、算数右移（有符号数）

逻辑右移：高位补0，低位溢出

eg：90 = 0101 1010 >>3

0000 1011 = 11

算数右移：高位补符号位，低位溢出（有符号数）

eg： -45 = 1010 1101>>3

-117 = 1111 0101

四、算数运算符

++i与i++（不同编译器结果不同）

int main() {
    int sum;
    int i  = 3;
    sum = (i++)+(i++)+(i++);
    printf("%d,%d\n",sum,i); // 12,6
    return 0;
}
int main() {
    int sum;
    int i  = 3;
    sum = (++i)+(++i)+(++i);
    printf("%d,%d\n",sum,i); // 16,6
    return 0;
}

6.变量的存储类别

1.内存分区

物理内存、虚拟内存

物理内存：实际存在的物理设备

虚拟内存：操作系统虚拟出来的内存。

操作系统在物理内存和虚拟内存中间做映射。

在32位系统中，每个进程的寻址范围为（4G）：0x00 00 00 00~0xff ff ff ff

应用程序中，我们使用的说虚拟地址。

虚拟内存分区

  1. 堆

    在动态申请内存时，在堆中开辟内存

  2. 栈

    主要存放局部变量

  3. 静态全局区

    1.未初始化的静态全局区

    静态变量（定义变量，static修饰），全局变量，没有初始化。

    2.初始化的静态全局区

    全局变量、静态变量、赋过初值、

  4. 代码区

    存放程序代码

  5. 文字常量区

    存放常量

2.普通的全局变量（函数体外）

int num = 100;

4.普通的局部变量

方法体或复合语句内有效

5.静态的局部变量

static + 局部变量

生命周期：第一次函数调用时：开辟内存空间，函数结束，不释放。一直使用之前的值。

默认：0

    return x;

int main() {

    printf("main:%d\n", add()); //5

八、预处理、动态库、静态库

1.C语言编译过程

  1. 预编译（不进行语法检查）

    将.c文件中的头文件展开，宏处理

    生成文件 .i文件

  2. 编译（进行语法检查）

    将预处理之后的文件 .i 文件 生成 .s汇编文件

  3. 汇编

    将.s汇编文件生成.o目标文件

  4. 链接

    将.o文件 链接程目标文件（可执行文件）

Linux下的GCC编译器编译过程

gcc -E 文件名.c   -o 文件名.i     //1.预处理
gcc -S 文件名.i   -o 文件名.s     //2.编译
gcc -c 文件名.s   -o 文件名.o     //3.汇编
gcc 文件名.o      -o 可执行程序名  //4.链接

2.include

include<> ：在系统指定的路径下寻找头文件（可以包含c文件，但是不建议，可能导致重复定义函数）

include “” ：在当前目录下寻找，找不到，再到系统指定的路径下寻找。

3.define（宏）

1.定义宏

宏 ：在预编译时进行替换（在后面的代码中出现了该标识符，那么就全部替换成指定的字符串）

#include <stdio.h>
#define Pi 3.14

int main() {
    printf("%f\n",Pi);
    
}

默认作用范围：当前文件下所有

使用#undef终止作用范围

#include <stdio.h>
#define Pi 3.14

int main() {
    printf("%f\n",Pi);
#undef Pi    //终止作用域
#define Pi 3.14159   //重新定义
    printf("%f\n",Pi);
}

2.带参宏

#define S(a,b) a*b

#include <stdio.h>
#define S(a,b) a*b

int main() {
    printf("%d\n", S(2,4));   // 8
    printf("%d\n", S(2+3,4)); // 2 + 3 * 4 = 14 
     //解决方法： #define S(a,b) (a)*(b)
     #define S(a,b) (a)*(b)
    printf("%d\n", S(2+3,4)); // 5*4 = 20
}

带参宏和函数的区别：

  1. 带参宏每次都会展开，函数每一次调用都会压栈
  2. 带参函数浪费时间，但节省空间

4.选择性编译（用于头文件重复）

#ifdef

#ifdef AAA

​    代码块1

#else 

   代码块2

#endif

如果当前c文件定义了AAA，编译代码块1，否则编译代码块2

#include <stdio.h>

int main() {

​    代码块1

5.静态库

1.动态编译（默认）

动态编译使用动态库文件进行编译

gcc  文件名.c -o 程序名

2.静态编译

静态编译使用静态库文件编译

gcc -static 文件名.c -o 程序名

3.两者区别

  1. 使用库文件不一致。

    静态编译使用静态库文件，动态编译使用动态库文件

注意

  1. 静态编译将静态库文件打包编译到可执行程序中

  2. 动态编译不会将动态库文件打包编译到可执行程序中

    它只是编译链接关系
